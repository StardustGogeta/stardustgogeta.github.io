Contributing - Style Guide

In order to maintain a high standard for the content on this website, it is necessary to set forth some rules for the styling of pages and their content. Please read everything below before proposing any changes on GitHub in order to minimize the amount of time necessary for revisions.

  1. Never plagiarize. This one should be obvious. If there is some ingenious result or clever proof to put here, cite where it came from. After stating the information, put a text link or end the sentence with a link in superscript, placing a space after the period. [src]
  2. Use the template. There is a file named TEMPLATE.html in this project's root directory. It is there for a reasonâ€”it includes the website stylesheet and math typesetting script necessary for writing a new page, and standardizes the format.
  3. Write TODOs. Do not leave a page unfinished or sorely lacking in detail without giving any indication. It is fine to have unfinished pages, unless they will never be improved upon. For this reason, put relevant TODOs on the bottom of the page.
  4. Use spacing properly. Sometimes, with math script and text together, it can be difficult to space things properly. Try to model new additions after existing pages and use the minimum amount of CSS possible.
  5. Write clearly. Egregious spelling or grammar errors are unacceptable. Similarly, obscure or poorly-typeset math can and will ruin a page. Use only standard math notation, and don't write anything contrived or unnecessarily complex.
  6. Include appropriate visuals. If a complex geometric argument must be made, construct a diagram in GeoGebra and render it on the page as a correctly-sized SVG image. Moreover, always typeset written mathematical expressions, do not leave them in plain text.
